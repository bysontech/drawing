project: party-lottery
cycle: 4
goal: 抽選体験を「かなり豪華」な演出へ強化し、主催者中心で見せ場になるアニメーション体験を作る

## Scope（Must）
1. 主催者向け抽選演出の大幅強化
- 抽選開始時にフルスクリーンまたは大きな表示領域で演出を再生する
- スロット風またはルーレット風の高速シャッフル演出を実装する
- 複数レイヤーのモーションを使い、背景・候補名・確定結果を段階的に見せる
- ranked=true の場合は 1位 → 2位 → 3位… と順番に見せ場を作る
- ranked=false の場合でも「停止→確定」の強い演出を入れる

2. 結果確定演出の強化
- 当選者確定時に拡大・発光・紙吹雪などの祝福演出を入れる
- role がある場合は、名前確定後に役割を遅れて表示する二段演出にする
- 結果カードをアニメーション込みで見やすく表示する
- draw済み結果の再表示時は、演出を簡略化して即表示も選べるようにする

3. 参加者向け演出の強化
- 結果公開ON時、参加者画面でも簡易版の豪華演出を見せる
- 自分が当選している場合は、自分の結果に特別な強調演出を入れる
- 非当選でも置いていかれないよう、全体結果または状態演出を表示する
- 公開OFF時は結果を出さず、雰囲気だけの待機/終了演出に留める

4. アニメーション設計の分離
- 抽選結果データとアニメーション進行状態を分離する
- 「結果はすでにDBにあるが、UI上は段階表示する」構造にする
- リロードや再訪時に演出をスキップして結果だけ見せられるようにする
- reduced motion や低性能端末へのフォールバックを用意する

5. 詳細設計の最適化
- 演出フェーズを state machine 的に定義する
- 主催者向け豪華演出と参加者向け簡易演出の責務を分ける
- ライブラリ依存を必要最小限に絞る
- 長すぎる演出でテンポを損なわないよう演出時間上限を定める

## Scope（Should）
- 音声再生の差し込みポイントを設計する（実装は任意）
- 背景エフェクトをテーマ別に差し替え可能にする
- 役割表示に専用モーションを入れる
- 抽選済み結果の再生ボタン（再演出）を主催者画面に用意する

## Out of Scope
- 抽選ロジック自体の変更
- WebSocket化
- 複数会場同期
- 動画書き出し
- 本格的なサウンド実装
- 3Dエンジン導入
- ログイン機能
- 品質フェーズのテスト拡張
- 本番運用監視の追加強化

## Detailed Design Notes
1. 演出アーキテクチャ
- サーバーは最終結果のみ返す
- フロントはその結果をもとに「未開始 → シャッフル → 停止 → 確定 → 祝福 → 次順位」の演出進行を持つ
- アニメーション進行状態は UI 専用 state として管理し、DBに保存しない
- リロード時は `drawn` 結果を即表示し、必要なら「再演出」ボタンでローカル再生する

2. 演出フェーズ定義
- `idle`
- `intro`
- `shuffle`
- `reveal_pending`
- `reveal_name`
- `reveal_role`
- `celebration`
- `completed`
- ranked=true の場合、winner ごとに `reveal_*` と `celebration` を繰り返す
- ranked=false の場合、短い shuffle 後に複数件まとめて reveal してもよい

3. UIレイヤー構成
- BackgroundLayer: 光・グラデーション・パルス
- MotionLayer: スロット/ルーレット/カード移動
- ResultLayer: 名前・順位・役割・確定表示
- FxLayer: 紙吹雪・グロー・フラッシュ
- OverlayLayer: カウントダウン、スキップ、再演出、ステータス

4. ライブラリ方針
- モーションは Framer Motion を第一候補
- 紙吹雪は軽量ライブラリまたは canvas ベースを採用
- ただし state 管理の中心は React 側に置く
- 依存追加は最小限にし、不要なUIライブラリは増やさない

5. UX方針
- 主催者画面は「会場スクリーンに映せる」見た目を優先する
- 参加者画面は主催者より控えめだが、自分が当選したときの喜びを強調する
- 1回の抽選演出は長くても 10〜15 秒程度に収める
- スキップ可能にし、テンポと見栄えの両立を図る

6. フォールバック方針
- reduced motion 時は演出を簡略化
- 低性能端末では背景エフェクト数を減らす
- 既存結果再訪時は即結果表示をデフォルトにする

## Definition of Done
- 主催者画面で豪華な抽選演出が再生される
- ranked on/off の両方で演出が成立する
- role がある場合に二段演出で表示される
- 参加者画面でも公開ON時に簡易演出が見える
- 当選参加者の自己強調演出が動く
- reduced motion または演出スキップ導線がある
- リロード後も結果表示が壊れない
- 少なくとも `build` が成功する
